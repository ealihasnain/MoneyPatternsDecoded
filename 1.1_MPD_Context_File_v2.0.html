AI Instructions & Guidelines
Standing rules that apply to every task — regardless of the specific prompt being used. These override any conflicting instruction embedded in a prompt.

Rule 0 — Evaluate Before Executing

Before performing any requested task, evaluate the alignment with channel goals (AVD, CTR, retention, brand voice). If a better angle, stronger hook, or higher-impact approach exists, propose it with one-line reasoning before proceeding. Never just comply — always add strategic value first.

RuleInstruction
Output Format All AI deliverables (scripts, prompts, upload packages) must be delivered inside a copyable script-box. Supporting information (word count, character count, rationale) is placed outside the box, never inside it.
Data Density Every script chapter must contain 3–6 quantitative data points. No chapter may run for more than 60 seconds without a specific number, statistic, or measurable comparison appearing on screen. This is non-negotiable.
ElevenLabs Split Scripts split into Part 1 and Part 2. Cut at the end of whichever complete chapter first crosses 4,500 characters. Never split mid-chapter or mid-sentence. Each part must be under 5,000 characters.
Anti-Padding Before adding a third example to any argument, ask: does this add a new layer of understanding, or only breadth? Breadth-only examples are cut. Each chapter covers exactly one emotional beat. Two beats in one chapter = split or cut.
No Financial Advice We explain mechanisms, not recommendations. Never write "you should invest in X" or "this is the best strategy." Always frame as systemic explanation: "here is how the mechanism works and why most people miss it."
Voiceover Spec ElevenLabs KateAsta voice (ID: UbWKKaYfYWMB1x0WtvN7). Every sentence tagged with at least one emotion tag. No two consecutive identical tags. Pauses on every sentence. [pause: 2.0s] used exactly once per script — at the single biggest reveal.
Channel Identity Always refer to the Channel Identity document uploaded in project files before any scripting task. Brand voice, pillar targeting, and visual system must be consistent throughout all outputs.
Quality Gate Run the 4-criteria alignment check before writing any script: (1) Hidden pattern? (2) Clear reframe moment? (3) Sufficient depth without padding? (4) No advice/get-rich angle? Score below 3/4 = flag and propose reframe before proceeding.
Icon Validity Icon concept names in the Chapters Table must be concrete nouns only — brand names, physical objects, named mechanisms. Abstract psychological concepts are invalid. Use visual equivalents instead: "calendar_missed" not "avoidance_brain".
Emotional Variation Scripts must contain distinctly different tag combinations at four mandatory moments: Chapter start (gear-shift signal), Chapter end bridge (forward tension), Reveal statement (maximum weight), Conclusion (earned warmth). These four must stand out from surrounding flow.

Retention Engineering
8 Techniques — Apply Per Video
🪝 Open Loop Hook
Plant a question in the first 10s that only resolves at Chapter 5+. Example: name a person or institution in the hook whose role only becomes clear at the reveal. Viewers stay to close the loop.
📌 Callback Numbers
Introduce one key number early (e.g. "$241/month") and re-invoke it repeatedly in new contexts — as debt payment, investment, emergency fund. Same number, compounding meaning. Creates continuity.
🌍 International Contrast
Show how other countries handle the same system. Makes the current system feel like a deliberate choice, not a law of nature. Creates a pattern interrupt that reframes everything the viewer assumed.
🧮 Live Calculation
Walk through actual math on screen: "Take your W-2, look at box 2..." Converts passive viewers to active participants. Most effective in Chapter 5–7 after credibility is established.
📈 The 30-Year Reveal
Show compound growth at 5y / 10y / 20y / 30y. The 30-year number is always deeply uncomfortable — that's where the screenshot moment is. Most people cap their mental projection at 3–5 years.
⚡ Villain Origin Story
The historical chapter: when was this mechanism created, by whom, and why? War-era designs that never went away. Institutional decisions framed as intentional choices. Lands across all political views.
🎯 Myth/Reality Pairs
Structure 2–3 beats as "What most people think" vs "What's actually happening." Split-screen format. Extremely shareable. High recall because of contrast framing. Can be a dedicated chapter mid-video.
🔔 The Action Moment
One chapter should be 60–90s of step-by-step: "Here's exactly what to do right now." Actionable endings are the #1 predictor of subscribe conversion on educational finance content.

Version Numbering Rules

Increment v# only when: (a) content materially changes after a review pass, (b) a bug is fixed that affects output integrity, or (c) the file has been re-exported with different data. Formatting-only tweaks do not warrant a version bump. Never reuse a version number — if v3 was broken, the fix is v4, never v3-final.

Topic Short Registry

Keep a running list of TopicShort slugs per week to prevent collisions. Slugs are PascalCase, ≤16 chars, and must be unique across all 46 weeks of the year. Example collisions to avoid: TaxRefund vs TaxReturn — use whichever is more specific and retire the other.

Section 05
Asset Library
Reusable visual assets built once and referenced across multiple weeks. All assets live in the Parchment Cinematic System (warm cream/linen background, Barlow Condensed headlines, teal accent, amber stat highlights). The library compounds — every week's new portrait, country card, brand badge, or chart template ships into this section via Step 3.10 and becomes available to subsequent weeks. Visual continuity across the 46-video calendar is one of the channel's strongest retention drivers; the library is the mechanism that delivers it.

Reusability Rule

Before drawing any new portrait, country card, brand badge, chart template, icon, or lower-third variant from scratch, check this registry. If the asset exists, reuse it verbatim — same stroke weights, same palette anchors, same dimensions. If it needs a variation, create a _v2 and document the delta in the Spec column. Never silently redesign an existing asset — that breaks visual continuity with older shipped videos and undoes the compounding advantage.

How The Library Compounds

Section 3.1 (Strategy) cross-references the library when proposing visual assets. Section 3.2 (Voiceover) tags chapter-table icons with library names. Section 3.6 (Video HTML) reuses library assets verbatim and emits a NEW ASSETS FOR LIBRARY manifest at the end of every Video HTML file. Section 3.10 (Asset Library Update) ingests that manifest and adds the new rows here. By Week 46, a P5 Psychology video can reference Friedman, Thaler, Kahneman, Tversky, Shiller, and Ariely portraits all from this library — none redrawn.

Section 06
Integrity Gates
Mandatory verification passes before any artifact moves to the next pipeline stage. Each gate has a specific pass/fail criterion. A failed gate halts the pipeline — no exceptions.

Gate Philosophy

MPD content is data-dense and fact-heavy. A single wrong number, misaligned caption, or unsourced claim erodes the trust the channel is built on. Gates exist to make verification mechanical, not optional. Run them every week, in order, without skipping.

G2 · Data Echo After Voiceover Script (3.2) Every number in the script appears exactly in the Strategy Brief. STEP 2A — TTS-SAFE RENDERING RULES (MANDATORY PRE-WRITE PASS)
══════════════════════════════════

These rules convert written text into TTS-renderable text. ElevenLabs v3 (KateAsta)
handles most prosody well, but only if the script avoids punctuation, formatting, or
numerals that trigger glitchy behaviour. Apply every rule below while drafting the
script. The downstream audio pipeline (crossfade join, LUFS normalisation, de-essing)
can clean up level and seam artefacts but CANNOT fix prosody, mispronunciations, or
runaway sentence length. Those must be solved in the script itself.

PUNCTUATION — FORBIDDEN INSIDE SPOKEN LINES:
— Ellipses "…" or "...". They trigger a long, uneven, often swallowed pause.
  Replace with a comma for a short beat or a period for a full stop. Pause tags
  do this job properly.
— Em-dashes and en-dashes inside a sentence. Replace with a comma, a period, or
  split into two sentences. (Dashes used as list bullets in THIS prompt are fine.
  The rule applies to SPOKEN LINES inside the scriptbox.)
— Semicolons. Always replace with a period. Engines treat them unpredictably.
— Parentheses. Rewrite as a separate sentence or integrate inline. Parens cause
  pitch drops that do not come back up.
— Quotation marks inside spoken lines unless you actually want "quote, end quote"
  prosody. Attribute with "According to X," or "X said," instead.
— More than TWO commas per sentence. Extra commas create stutter-pauses.
— Bare text without a terminal mark. Every sentence ends with a period, question
  mark, or exclamation. The engine needs the mark to release the final pitch
  contour cleanly.

NUMBERS — SPELL OUT EVERY ONE IN THE SPOKEN LINE:
— Years. "1999" becomes "nineteen ninety-nine". "2024" becomes "twenty twenty-four".
  Never leave digits for a year.
— Large numbers. "1,400,000" becomes "one point four million". "847" becomes "eight
  hundred forty-seven" or "roughly eight hundred fifty" depending on register.
— Decimals use "point" not "dot". "3.14" becomes "three point one four".
— Currency spells both parts. "$1.2 billion" becomes "one point two billion dollars".
  Never leave a bare "$" symbol in a spoken line.
— Percentages use the word "percent". "47%" becomes "forty-seven percent".
— Dates expand. "Jan 3, 1942" becomes "January third, nineteen forty-two".
— Ranges use "to" not a dash. "1939–1945" becomes "nineteen thirty-nine to nineteen
  forty-five".
— Ordinals. "3rd" becomes "third". "21st" becomes "twenty-first".
— Units. First occurrence spell out. "5 kg" becomes "five kilograms". "25°C" becomes
  "twenty-five degrees Celsius".

IMPORTANT CONTRAST WITH UPPERCASE RULES BELOW: the UPPERCASE RULES in Step 3 apply
to SPELLED-OUT WORDS only. "THIRTY-ONE BILLION dollars" is valid. "31 BILLION DOLLARS"
is not. Spell first, uppercase second.

ACRONYMS — FORCE PRONUNCIATION EXPLICITLY:
— Spelled-letter acronyms get periods plus spaces. IRS becomes "I. R. S.". NSA
  becomes "N. S. A.". W-2 becomes "W. two". 401(k) becomes "four oh one k".
— Word-pronounced acronyms stay as written. NATO, NASA, FICA, FAFSA. Test once,
  then trust.
— First use always expand. "The Internal Revenue Service, or I. R. S., ..."
  Afterwards "I. R. S." is enough.
— Single letters in names space-separated. "J. P. Morgan". "F-35" becomes "F
  thirty-five".

PROPER NOUNS — VERIFY AND RESPELL IF NEEDED:
— Every proper noun on first appearance: confirm KateAsta pronounces it correctly.
  If not, respell phonetically IN THE SCRIPT. Example: "Xi Jinping" becomes
  "Shee Jin-ping" if the engine butchers it. The on-screen caption keeps the
  correct spelling — the two can diverge.
— Place names follow the same rule. "Worcestershire" becomes "Wooster-shire".
  "Reykjavik" becomes "Ray-kya-veek".
— Recurring names (economist names, company names, country names) must pronounce
  identically every time, within a video and across the series. Maintain a
  project glossary if unsure.

HOMOGRAPHS — DISAMBIGUATE BY REWRITE:
Same spelling, different pronunciation. KateAsta picks one based on context —
sometimes the wrong one. Rewrite to remove ambiguity. Common offenders and fixes:
  lead (metal) versus lead (to guide)        → use "the element lead" or rewrite
  read (present) versus read (past)          → use "has read" or "is reading"
  tear (from eye) versus tear (to rip)       → rewrite
  bass (fish) versus bass (instrument/low)   → rewrite
  wind (air) versus wind (to turn)           → rewrite
  minute (time) versus minute (small)        → rewrite
  desert (sand) versus desert (to abandon)   → rewrite
  live (verb) versus live (adjective)        → rewrite

SENTENCE LENGTH — ENFORCE CAP:
— Target: 12–20 spoken words per sentence.
— HARD CAP: 28 words. Beyond 28, the engine runs out of prosodic breath and the
  pitch flatlines in the second half. No exceptions.
— Alternate long and short. After a long sentence, follow with a short one. This
  is the single biggest natural-delivery driver.
— No nested subclauses. "The ship, which had been built in nineteen twelve and
  refitted twice, sank" becomes two sentences.
— Read every sentence aloud during the draft. If you run out of breath, so will
  the engine.

FILLERS AND VOCAL TICS — NEVER WRITE:
No "um", "uh", "hmm", "well,", "you know". The engine reads them literally and it
sounds robotic in almost every case. If you want a hesitant or thoughtful register,
use sentence structure (short clause, period, short clause). Never transcribed tics.

EMPHASIS — ELEVENLABS V3 EXCEPTION:
ALL CAPS for emphasis IS permitted on ElevenLabs v3 specifically. The generic TTS
guideline "never use ALL CAPS" applies to engines that spell caps letter by letter.
v3 handles capitalized words as emphasis without spelling them out. KEEP the
UPPERCASE RULES from Step 3 unchanged. Never use asterisks, underscores, or
markdown — those get read aloud.

Rules:
— Entries cover full 0.0–1.0 range, no gaps
— 3–5 phrases per chapter
— BIG TEXT: ALL CAPS, 2–5 words, EXTRACTED from SRT_TIMELINE.

  SRT words or a paraphrase that uses ONLY words spoken in that
  chapter. No new vocabulary.

  the spoken span this phrase visualises. progress_start derives

  forbidden — they are computed.
— Fade in over first 12% of range, fade out over last 12%
— BANNED phrasings (these are AI slop, never use): "the truth is",
  "what they don't tell you", "here's the catch", "did you know",
  "studies show", "experts agree", generic motivational fragments.
  If the phrase isn't in the SRT, it doesn't exist.

  Big text:  Barlow Condensed 900, sz = clamp(H*.072, 34, 90)

             color: C.gray or C.ink2

SECTION 4 — DATA VISUALISATION ANIMATIONS
═══════════════════════════════════════════

This is the critical layer. Since scripts contain 80+ data points across 9 chapters,
each chapter animation must display data — not just abstract patterns.
Minimum 5–7 verified or estimated data points per chapter must be visually represented.

TRANSITION SPEED PHILOSOPHY — NON-NEGOTIABLE:
Graphics and data sets must transition at professional editing pace — not slow-spread.
The rule is: transition fast, then hold while relevant captions are on screen.

TRANSITION TIMING RULES:
  — Animate IN:  every graphic element enters in 0.25–0.45s (spring or ease-out). NEVER over 0.6s.
  — HOLD:        once fully rendered, the graphic holds at full opacity for the duration of its
                 associated SRT caption block(s). The hold is determined by iconTimes[] — the graphic
                 stays visible until the next icon trigger fires, or until chapter end.
  — Animate OUT: fade or slide out in 0.2–0.3s when the next graphic enters.
  — Sequential elements (bars, timeline nodes, flow boxes): stagger entry by 120–180ms max.
    Do not space stagger beyond 200ms — it reads as broken, not dramatic.
  — Stat slams: spring-in over 18–22 frames at LOOP_FPS. Full opacity by frame 22. Hold until
    the next icon trigger. Do not cycle or fade a stat while its caption is still on screen.
  — Growth curves: draw left-to-right over 0.5s maximum, then hold at full draw.
    Do not time the draw to stretch over the full chapter — draw it, then let it sit.
  — Comparison bars: all bars fill simultaneously over 0.4s. Hold.
  — Timeline nodes: animate in left-to-right with 150ms stagger. Each node snaps in (not floats in).
  — Myth/Reality rows: each row pair animates in with 180ms stagger. Strikethrough fires at prog>0.60
    in 0.2s. Hold.
  — Flow diagram boxes: spring in with 160ms stagger. Connector arrows draw in 0.15s.
  — Hold frames (share-bait): enter in 0.3s, hold for minimum 4 seconds, no animation during hold.

4B-1 · COUNT-UP NUMBERS (numStat)

RULE: Every numeric stat slam MUST use numStat(), never stat() directly.

  — Text-only stats (FIXABLE, ZERO, ONLY, words): use stat() as normal — no count-up

4B-2 · AMBIENT PARTICLES

PURPOSE:

  constant motion that signals "this is video, not a slide." They are intentionally
  very subtle — opacity 0.024–0.057, radius 1.2–3px. Viewers don't consciously

  // Deterministic seeds — no Math.random() (breaks frame-accurate export)
  const PARTS = (function() {
    const s = [0.137,0.239,0.341,0.431,0.512,0.619,0.724,0.813,0.907,
               0.178,0.056,0.462,0.881,0.325];
    return s.map((v, i) => ({
      ox:  ((v*7.3   + s[(i+3)%14]*.8) % 1),
      oy:  ((s[(i+5)%14]*5.1 + s[(i+7)%14]*.6) % 0.88),
      r:   1.2 + (i % 4) * .55,
      spd: 0.007 + (i % 5) * .0035,
      ang: (i * 47.11) % (Math.PI * 2),
      al:  0.024 + (i % 4) * .011
    }));
  })();

  function drawParticles() {
    const t = currentSec;
    ctx.save();
    PARTS.forEach(p => {
      const px = ((p.ox + Math.cos(p.ang) * p.spd * t) % 1 + 1) % 1;
      const py = ((p.oy + Math.sin(p.ang) * p.spd * t) % 0.88 + 0.88) % 0.88;
      ctx.beginPath(); ctx.arc(px * W, py * H, p.r, 0, Math.PI * 2);
      ctx.fillStyle = C.ink; ctx.globalAlpha = p.al; ctx.fill();
    });
    ctx.globalAlpha = 1; ctx.restore();
  }

CHAPTER FADE IMPLEMENTATION — copy exactly, do not inline or approximate:

  const FADE_DUR = 5; // seconds each side
  function drawChapterFade(ci) {
    const ch = CHAPTERS[ci];
    const chDur = ch.end - ch.start;
    const chSec = clamp(currentSec - ch.start, 0, chDur);
    const fadeIn  = clamp(chSec / FADE_DUR, 0, 1);
    const fadeOut = clamp((chDur - chSec) / FADE_DUR, 0, 1);
    const overlayAlpha = 1 - Math.min(fadeIn, fadeOut);
    if (overlayAlpha <= 0) return;
    ctx.save();
    ctx.globalAlpha = overlayAlpha;
    ctx.fillStyle = '#1A1612';
    ctx.fillRect(0, 0, W, H);
    ctx.globalAlpha = 1;
    ctx.restore();
  }

RULE: drawChapterFade() MUST appear after drawVignette() in BOTH render() AND the export
  loop. It is the only draw call that supersedes the vignette. Never move it earlier.

Gemini Rendering Lessons — Hard-Won Rules (encode into every prompt)

These rules were established through iteration. Each one fixes a known Gemini failure mode. Every generated Gemini prompt must incorporate all of them — do not revert to shorter or friendlier versions.
